# Módulo 6: Proyecto CRUD Completo (Catálogo de Productos)

¡Llegó el momento de la verdad! En este módulo vamos a dejar la teoría aislada a un lado y construiremos una aplicación real desde cero: **Un Catálogo de Productos / Inventario**. 

Aplicarás todo lo aprendido en los módulos anteriores (Rutas, Vistas, Controladores, Base de Datos, Formularios y Archivos) para crear un panel de administración profesional donde podrás gestionar categorías y productos.

## 📖 Temas del Módulo

1. **[Tema 1: Planificación y Base de Datos](./01-base-de-datos/README.md)** Diseña el esqueleto del proyecto. Aprende a pensar en la estructura de datos y crea las migraciones para las categorías y los productos.

2. **[Tema 2: Modelos, Relaciones y Seeders](./02-modelos-y-seeders/README.md)** Conecta tus tablas y genera decenas de productos de prueba al instante para no tener que diseñar con una tienda vacía.

3. **[Tema 3: Rutas y el Controlador de Recursos](./03-rutas-y-controlador/README.md)** Configura el "cerebro" de tu aplicación usando controladores RESTful.

4. **[Tema 4: Layout Base, Listado y Paginación](./04-listado-y-paginacion/README.md)** Crea la plantilla maestra visual de la tienda y descubre cómo usar la magia de la paginación de Laravel para no saturar la pantalla.

5. **[Tema 5: Formulario de Creación y Subida de Imágenes](./05-creacion-y-archivos/README.md)** Diseña una interfaz limpia para añadir productos nuevos, validando precios, stock y guardando la foto del producto en el servidor.

6. **[Tema 6: Edición y Eliminación](./06-edicion-y-eliminacion/README.md)** Cierra el ciclo de vida de los datos permitiendo actualizar la información y borrar productos de forma segura.

7. **[Tema 7: Barra de Búsqueda y Filtros](./07-busqueda-y-filtros/README.md)** *(Avanzado)* Dale superpoderes a tu catálogo permitiendo buscar productos por nombre.

8. **[Tema 8: Soft Deletes (La Papelera de Reciclaje)](./08-soft-deletes/README.md)** *(Avanzado)* Descubre el secreto de las empresas reales: cómo borrar registros sin eliminarlos de la base de datos para no perder el historial.
